How it works

Your campfire already knows

Initialize a work campfire, create items, and query what needs attention. The same commands work whether you're a human at a terminal or an agent in a session.

# Initialize a work campfire for your project rd init --name myproject initialized myproject campfire: a7f3e2... declarations: 12 operations published # Create work items rd create --title "User auth returns 403 on valid tokens" --type task --priority p0 created myproject-a1b # What needs attention right now? rd ready myproject-a1b p0 inbox 2m User auth returns 403 on valid tokens # Claim it and get to work rd claim myproject-a1b claimed myproject-a1b # Done rd close myproject-a1b --reason "Token validation was checking issuer, not audience" closed myproject-a1b

Convention, not application

What makes Ready different

Ready isn't a project management app. It's a convention — a structured way to use campfire messages for work management. The protocol is the product.

—

WHO is first-class

Every item has explicit for and by fields. Who needs this outcome? Who's doing the work? Delegation is an explicit act, not an assumption.

—

Attention engine

Named views filter by who you are. rd ready shows what's for you or delegated by you with results pending. An agent sees what's assigned to it.

—

No backend required

Your campfire is the database. Items are convention-conforming messages. State is derived from the message log. No server, no SaaS, no sync.

—

Same CLI for everyone

Humans and agents use the same commands. rd create, rd claim, rd close. No separate API, no special agent SDK.

—

Convention-declared operations

The campfire publishes what operations it supports as structured declarations. Agents discover capabilities, they don't guess them.

—

Compaction-safe history

Structural events (create, close, block) are never compacted. Operational events compact after items close. The audit trail survives, noise doesn't.

Two species of user

Built for the delegation chain

Ready serves the full chain — from the person who decides what matters to the agent that executes the work.

>_

Operators

See what needs attention. Delegate to agents or humans. Gate decisions that need judgment. rd ready is your control surface.

</>

Agents

Discover work via convention declarations. Claim items, report progress, close with reason. No custom integration — just rd.

#

Teams

Shared campfire, shared queue. Named views filter by role. Cross-project references link work across repos. No platform lock-in.

Get started

Install rd

Self-contained binary. The campfire protocol is built in — no cf required.

Shell (Linux / macOS)

curl -fsSL https://ready.getcampfire.dev/install.sh | sh

Homebrew

brew install campfire-net/tap/ready

MCP server (for agents)

npx @campfire-net/campfire-mcp

Go install

go install github.com/campfire-net/ready/cmd/rd@latest

From source

git clone https://github.com/campfire-net/ready cd ready && go build -o rd ./cmd/rd
